# HealthCare Dev List

## 2026-03-23

이번 채팅에서 진행한 개발 내역을 간단히 정리합니다.

- `Android/` 안드로이드 앱 프로젝트를 기준으로 UI/UX를 전면 정리
- Android 7.1.2 기준 자동 서버 연결, 전체화면 immersive 모드 적용
- 초기 대기 화면, 얼굴 스캔 화면, 전신 스캔 화면으로 3페이지 흐름 구성
- 페이지 전환 시 가로 슬라이드 애니메이션 적용
- 얼굴 페이지에서 카메라 스트림 자동 연결 및 얼굴 가이드 프레임 표시
- 전신 페이지 추가 및 얼굴 페이지의 중앙 버튼으로 이동되도록 연결
- `old/BodyCheck.py`의 포즈 인식 흐름을 참고해 전신 스켈레톤 표시 기능 반영
- 전신 페이지에서 카메라 원본 위에 스켈레톤이 함께 보이도록 백엔드 확장
- 전신 페이지의 로딩 상태 문구, 카드 배치, 버튼/하단 정보 영역 겹침 문제 수정
- 실기기 기준으로 전신 페이지 안내 카드와 하단 요소 정렬 문제 반복 보정
- `BackEnd/`를 FastAPI 기준으로 정리하고 불필요한 예전 잔여 파일 제거
- FastAPI에 `/health`, `/offer`, `/mjpeg`, `/body-skeleton.mjpeg` 경로 구성
- 카메라 해상도는 4K 우선 요청, 지원 범위 안에서 최적 해상도로 fallback 되도록 유지
- `BackEnd/worker/requirements.txt`에 `mediapipe` 의존성 반영
- 안드로이드 앱 재빌드 및 서버 재시작, 헬스체크와 전신 스트림 응답 확인 완료

## 현재 기준

- 안드로이드 앱은 로컬 핫스팟/동일 네트워크 기준 자동 접속 구조
- 백엔드는 `BackEnd/main.py` 실행 기준
- 기본 서버 포트는 `8080`
- 최근 실행 기준 카메라는 `1번 카메라`
- 전신 페이지는 `카메라 원본 + 스켈레톤 오버레이` 방식
