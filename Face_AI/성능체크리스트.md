공개 체크포인트가 실제로 의미 있게 학습되었는지 보기 위해, 각 과제에서 가장 많은 클래스를 고르는 단순 베이스라인과 비교함.

| 항목 | 원모델 tolerant accuracy | 다수 클래스 tolerant accuracy | 차이 |
|---|---:|---:|---:|
| dryness | 0.971963 | 0.971963 | 0.000000 |
| pigmentation | 0.836813 | 0.682243 | +0.154570 |

해석:

- `pigmentation` 은 모델이 실제로 이김

- `pore`, `wrinkle` 는 오히려 다수 클래스보다 약함
- 평균도 다수 클래스보다 약간 낮음

## 9. 원본 대비 모델 성능 퍼센트에 대한 결론

`원본 모델 성능을 100으로 놓고 현재가 몇 %인지`는 현재 자료만으로는 계산 불가함.

이유:

- 공식 AI Hub 페이지와 공개 문서에 `원본 모델의 공식 benchmark 수치`가 없음
- 현재 확인 가능한 것은 `로컬 reference 기준 절대 성능`뿐임

즉:

- `원본 대비 데이터 보유율`은 계산 가능
- `원본 대비 성능 퍼센트`는 공식 원본 성능 부재로 정확 계산 불가

## 10. 성능이 떨어지는 핵심 이유

### 10-1. 가장 큰 원인: 심한 클래스 불균형

reference 전체 라벨 분포를 `TL.zip + VL.zip` 기준으로 전부 집계한 결과, 희소 클래스가 매우 부족함.

대표 사례:

| 항목 | 다수 클래스 | 희소 클래스 |
|---|---|---|
| chin_sagging | class 0 = 5,720 | class 6 = 13 |
| forehead_pigmentation | class 1 = 6,422 | class 5 = 52 |
| lip_dryness | class 2 = 7,631 | class 0 = 273, class 4 = 286 |
| l_cheek_pore | class 2 = 7,579 | class 5 = 130 |
| r_cheek_pore | class 2 = 7,670 | class 5 = 130 |

결과:

- 모델이 희소 등급을 충분히 배우기 어려움
- 다수 클래스만 찍어도 점수가 잘 나오는 구조가 생김
- 특히 `wrinkle`, `sagging`, `pore` 의 희귀 고등급이 약해짐

### 10-2. 현재 로컬 데이터가 원본 전체가 아님

| 축 | 상태 |
|---|---|
| 이미지 | 원본 대비 약 90% |
| 라벨 | 원본 대비 약 90% |
| 피실험자 | 원본 대비 약 90% |
| 측정 데이터 | 원본과 동일 |
| 메타데이터 | 원본과 동일 |

결과:

- 공개 체크포인트와 현재 로컬 reference가 완전히 같은 조건이 아님
- 공식 전체본 기준의 일반화 성능을 그대로 재현하기 어려움

### 10-3. 런타임 문제는 일부 있었지만 핵심 원인은 아니었음

수정한 내용:

- split patch 방향 처리 불일치 정리
- 라이브 오버레이 한글 렌더링 수정
- reference 기반 하이브리드 보정 모드 추가

다만 split patch 수정만으로는 표본 200장 기준 성능 변화가 거의 없었음.

| 구분 | dryness | pigmentation | pore | sagging | wrinkle | 평균 |
|---|---:|---:|---:|---:|---:|---:|
| 수정 전 | 0.895000 | 0.871667 | 0.787500 | 0.790000 | 0.554557 | 0.779745 |
| 수정 후 | 0.895000 | 0.870000 | 0.787500 | 0.790000 | 0.554557 | 0.779411 |

결론:

- 런타임 버그는 보조 원인
- 근본 원인은 `데이터 분포`와 `공개 체크포인트 한계`

## 11. 실사용 성능 보정 결과

reference 분포를 활용해 `모델이 실제로 강한 항목은 모델값`, `계속 약한 항목은 기준 분포 보정값`을 쓰는 `reference_hybrid` 모드를 추가함.

표본 200장 기준 비교:

| 구분 | dryness | pigmentation | pore | sagging | wrinkle | 평균 |
|---|---:|---:|---:|---:|---:|---:|
| 원모델 | 0.895000 | 0.870000 | 0.787500 | 0.790000 | 0.554557 | 0.779411 |
| 하이브리드 보정 | 0.895000 | 0.870000 | 0.790000 | 0.790000 | 0.739409 | 0.816882 |

해석:

- `wrinkle` 가 가장 크게 개선됨
- `pore` 도 소폭 개선
- `dryness`, `pigmentation`, `sagging` 은 거의 유지
- 평균은 `0.779411 -> 0.816882` 로 상승

## 16. 재학습 방향

### 데이터 준비

1. training 자산까지 모두 정리

```powershell
.\.venv\Scripts\python.exe Face_AI\executable\prepare_assets.py --include-training
```

2. 공식 `Training` 과 `Validation` 은 분리 유지
3. 추가 분할이 필요하면 피실험자 기준으로만 분할

### 학습 전략

1. 영역별 모델 구조는 공식 공개 구조 유지
2. class-aware sampling 추가
3. weighted cross-entropy 또는 focal loss 적용
4. 희소 등급 우선 oversampling
5. exact accuracy, tolerant accuracy, per-class recall 모두 기록

### 검증 정책

반드시 같이 봐야 할 지표:

| 지표 | 의미 |
|---|---|
| exact accuracy | 완전 일치 정확도 |
| tolerant accuracy | `abs(pred - gt) < 2` 허용 정확도 |
| per-class recall | 희소 클래스 복원 능력 |
